3.5
path
3.5.1
Description
1
Values of the path type represent valid paths.
3.5.2
Definition
1
A string s can be interpreted as a value of type path iff s can be split into a sequence of valid path components separated by single forward slash "/" (U+002F) characters. A valid path component is defined by the valid_path_component predicate:
3.5.2.1. Type predicate (valid_path_component)
valid_path_component : string → boolean =
  ∀t. (empty(t) = false) ∧ (contains(t,"..") = false),
    ∀i. in(t, i), 
      ∀c. c = get(t,i),
        is_digit(c) ∨ is_letter(c) ∨ (c = ".") ∨ (c = "_") ∨ (c = "-")
2
A valid path is required to begin with a single forward slash "/" (U+002F), and must be longer than a single character. If s is divided into substrings delimited by U+002F (such that U+002F does not appear in any resulting substring), then for each substring t, valid_path_component(t) = true.
3.5.3
Java
1
In the io7m-jfprop server implementation, the path type is represented by the JFPRepositoryPath type.
