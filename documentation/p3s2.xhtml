3.2.1
length
3.2.1.1. Specification
length : ∀t. list[t] → integer
1
The length function returns the number of values in the given argument list.
3.2.2
empty
3.2.2.1. Specification
empty : string → boolean
1
The empty function returns true for a value s iff length(s) = 0.
3.2.3
is_digit
3.2.3.1. Specification
is_digit : character → boolean
1
The is_digit function returns true iff the given category of the argument is DECIMAL_DIGIT_NUMBER.
3.2.4
is_letter
3.2.4.1. Specification
is_letter : character → boolean
1
The is_letter function returns true iff the given category of the argument is one of UPPERCASE_LETTER, LOWERCASE_LETTER, TITLECASE_LETTER, MODIFIER_LETTER, or OTHER_LETTER.
3.2.5
in
3.2.5.1. Specification
in : ∀t. (list[t], integer) → boolean
1
The in function returns true for arguments (k, i) iff i is greater than or equal to 0 and less than length(k).
3.2.6
get
3.2.6.1. Specification
get : ∀t. (list[t], integer) → character
1
The get function called with arguments (k, i) returns the value in the list k at index i. The results are undefined if in(k, i) = false.
3.2.7
contains
3.2.7.1. Specification
contains : ∀t. (list[t], list[t]) → boolean
1
The contains function called with arguments (k, m) returns true if the list k contains the sequence of values given by list m.
