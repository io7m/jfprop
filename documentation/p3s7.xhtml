3.7
time_component_spec
3.7.1
Description
1
Values of the time_component_spec type represent patterns for matching periods of time. A value of the time_component_spec type can be of one of three forms:
3.7.1.1. Type definition
  1. any - matches any value.
  2. exact n - matches values that are equal to n.
  3. divisible n - matches values that are divisible by n.
2
Values of the time_component_spec are usually combined to specify times. The io7m-jfprop uses three values of type time_component_spec to specify the times of day that mass synchronizations will be executed, specifying the day, hour, and minute, respectively. As an example, (any, divisible 2, exact 30) specifies that a synchronization should run on all days, on even-numbered hours, at 30 minutes past the hour.
3.7.2
Java
1
In the io7m-jfprop server implementation, the time_component_spec type is represented by the JFPTimeComponentSpecType type.
